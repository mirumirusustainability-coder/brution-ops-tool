---

## 3. AI 모델 전략 (Advisor Strategy)

| 직급 | 모델 | 역할 |
|---|---|---|
| 부장급 | Claude Opus 4.6 | 최종 검수/승인, 전략 판단 (Advisor, max_uses: 3) |
| 과장/차장급 | Claude Sonnet 4.6 | 실무 분석/초안 작성 (Executor, 주력) |
| 사원/대리급 | Claude Haiku | 데이터 처리/분류/파싱 |

**품질 보증 구조:**
```
Haiku (데이터 처리)
    ↓
Sonnet (실행 주도 + Opus 어드바이저 탑재)
    ↓ 복잡한 전략 판단 필요시만 Opus 호출
담당자 검토 (보강/재지시 가능)
    ↓
published → 고객사 전달
```

---

## 4. 브루션 5단계 프로세스

| STEP | 라벨 | 브루션 업무 |
|---|---|---|
| 0 | 스타터 패키지 | 시장조사/키워드/경쟁사/샘플분석 |
| 1 | 브랜드 기획 | BI/시장조사확장/제품후보확정 |
| 2 | 디자인·인증 | 제품디자인/패키징/인증(유동적) |
| 3 | 생산·납품 | QC/포워딩/통관/납품 |
| 4 | 운영지원 | 마케팅/성과분석/판매전략 |

## 5. 핵심 데이터 흐름

```
고객사 템플릿 입력
    ↓
브루션 AI 분석 (Sonnet + Opus Advisor)
    ↓
담당자 검토/승인
    ↓
고객사 전달 (published)
    ↓
마켓 연동 성과 추적
    ↓
월간 리포트 자동화
```

```

... 버튼 → 드롭다운 메뉴:
  ├─ 프로필 수정
  ├─ 프로필 이미지 수정
  ├─ 계약 내용 수정
  ├─ 브랜드 정보 수정
  ├─ 단축 기능 설정 (최대 4개 커스텀)
  └─ 고객사 삭제

단축 기능 바 (커스텀 가능, 기본: 채팅/견적서/승인요청/리포트)
상태 뱃지 (진행중/D-day/패키지)

기본 정보:
  ├─ 고객사 담당자 / 연락처 / 이메일
  ├─ 홈페이지 URL
  ├─ 스마트스토어 URL
  └─ 쿠팡 스토어 URL

계약 정보:
  ├─ 패키지 / 계약금액 / 계약기간 / 만료일
  ├─ 판매채널 / 사업자번호
  └─ (만료일 D-30 이내 빨강 강조)

브랜드 정보:
  ├─ 브랜드명 / 타겟 / 톤앤매너
  └─ 금지 방향

AI 크레딧:
  ├─ 월간 한도 / 사용량 / 잔여량
  ├─ 기능별 소비량
  └─ 리셋 날짜

브루션 담당:
  ├─ 담당자 / 정기 미팅 / 마지막 소통
```
